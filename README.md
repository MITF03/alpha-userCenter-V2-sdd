# 會員中心 SDD

### 語法

/constitution - 建立基本的規範(憲法)

/specify - 描述你想建立的專案樣式，這邊無關技術，就只是單純的說明想做什麼跟為什麼要做

/contracts - api 設計(?)

/plan - 說明這個專案有需要用到的技術跟希望的技術架構

/tasks - 將任務細分

/implement - 執行

### 優點

- AI 會想到一些我們平常可能會漏掉的問題 (Edge Cases )
- quickstart 寫得很完整，讓人可以很容易就開啟一個新的專案
- research 把每個套件的用途跟優缺點都列出來，這樣讓在選擇技術選型的團隊有個更好的選擇依據

### 缺點

- 中英參雜的很奇怪 (例如: when...then)
- 分類方式感覺不夠客製化 會針對你下的指令去分類(/spec 一個指令會是一個 folder)
- 步驟方面有點太 detail(雙面刃)

```
/ frontend
    / 001
    / 002
    / 003

/ backend
    / 001
    / 002
    / 003
```
