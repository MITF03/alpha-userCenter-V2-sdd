## User Scenarios & Testing _(mandatory)_

### User Story 1 - 側邊欄導航顯示 (Priority: P1)

使用者需要能夠在應用程式中看到側邊欄導航，並透過側邊欄存取各個功能。

**Why this priority**: 側邊欄是應用程式的主要導航方式，必須先建立才能讓使用者存取其他功能。這是所有功能入口的基礎。

**Independent Test**: 可以透過以下方式獨立測試：使用者進入應用程式，看到側邊欄顯示所有功能入口，點擊任一功能入口能夠導向對應頁面。此功能可獨立運作並提供價值，即使個別功能頁面尚未完成。

**Acceptance Scenarios**:

1. **Given** 使用者進入應用程式, **When** 查看側邊欄, **Then** 系統顯示所有功能入口：開戶流程、我的計劃、投資健檢、學習專區、交易查詢、個人資料、常見問題
2. **Given** 使用者查看側邊欄, **When** 點擊「開戶流程」, **Then** 系統導向首頁/開戶流程頁面
3. **Given** 使用者查看側邊欄, **When** 點擊「我的計劃」, **Then** 系統導向我的計劃頁面
4. **Given** 使用者查看側邊欄, **When** 點擊「投資健檢」, **Then** 系統導向投資健檢頁面
5. **Given** 使用者查看側邊欄, **When** 點擊「學習專區」, **Then** 系統導向學習專區頁面
6. **Given** 使用者查看側邊欄, **When** 點擊「交易查詢」, **Then** 系統導向交易查詢頁面
7. **Given** 使用者查看側邊欄, **When** 點擊「個人資料」, **Then** 系統導向個人資料頁面
8. **Given** 使用者查看側邊欄, **When** 點擊「常見問題」, **Then** 系統導向常見問題頁面
9. **Given** 使用者正在某個功能頁面, **When** 查看側邊欄, **Then** 系統高亮顯示當前所在的功能入口

---

### User Story 2 - 我的計劃功能 (Priority: P2)

使用者需要能夠查看和管理自己的投資組合，這些投資組合顯示在「我的計劃」功能中。

**Why this priority**: 這是使用者查看自己投資狀況的核心功能，提供個人化的投資組合管理。雖然重要，但可以在側邊欄導航建立後再實作。

**Independent Test**: 可以透過以下方式獨立測試：使用者點擊側邊欄的「我的計劃」，進入我的計劃頁面，查看自己的投資組合列表，可以查看各個投資組合的詳細資訊。此功能可獨立運作，提供完整的投資組合查看價值。

**Acceptance Scenarios**:

1. **Given** 使用者已登入, **When** 點擊側邊欄「我的計劃」, **Then** 系統顯示使用者的投資組合列表
2. **Given** 使用者在我的計劃頁面, **When** 查看投資組合列表, **Then** 系統顯示每個投資組合的基本資訊（名稱、總價值、投資標的數量等）
3. **Given** 使用者在我的計劃頁面, **When** 點擊特定投資組合, **Then** 系統顯示該投資組合的詳細資訊
4. **Given** 使用者有多個投資組合, **When** 查看我的計劃頁面, **Then** 系統以列表或卡片形式顯示所有投資組合
5. **Given** 使用者尚未建立任何投資組合, **When** 查看我的計劃頁面, **Then** 系統顯示空狀態提示，引導使用者建立第一個投資組合

---

### User Story 3 - 投資健檢功能 (Priority: P2)

使用者需要能夠進行投資健康檢查，評估自己的投資狀況與風險。

**Why this priority**: 投資健檢幫助使用者了解自己的投資狀況，提供風險評估與建議。這是重要的輔助功能，但可以在核心功能完成後再實作。

**Independent Test**: 可以透過以下方式獨立測試：使用者點擊側邊欄的「投資健檢」，進入投資健檢頁面，完成健檢問卷或輸入投資資訊，系統顯示健檢結果與建議。此功能可獨立運作，提供完整的健檢價值。

**Acceptance Scenarios**:

1. **Given** 使用者已登入, **When** 點擊側邊欄「投資健檢」, **Then** 系統導向投資健檢頁面
2. **Given** 使用者在投資健檢頁面, **When** 完成健檢問卷或輸入投資資訊, **Then** 系統計算並顯示健檢結果
3. **Given** 使用者完成投資健檢, **When** 查看健檢結果, **Then** 系統顯示風險評估、投資建議、改善建議等資訊
4. **Given** 使用者已完成過投資健檢, **When** 再次進入投資健檢頁面, **Then** 系統顯示歷史健檢記錄與最新結果

---

### User Story 4 - 學習專區功能 (Priority: P3)

使用者需要能夠存取學習資源，提升投資知識與技能。

**Why this priority**: 學習專區提供教育資源，幫助使用者學習投資知識。雖然有價值，但可以在其他核心功能完成後再實作。

**Independent Test**: 可以透過以下方式獨立測試：使用者點擊側邊欄的「學習專區」，進入學習專區頁面，瀏覽學習資源（文章、影片、課程等），選擇並查看學習內容。此功能可獨立運作，提供完整的學習體驗。

**Acceptance Scenarios**:

1. **Given** 使用者進入應用程式, **When** 點擊側邊欄「學習專區」, **Then** 系統導向學習專區頁面
2. **Given** 使用者在學習專區頁面, **When** 瀏覽學習資源, **Then** 系統顯示各種學習內容（文章、影片、課程等）
3. **Given** 使用者在學習專區頁面, **When** 點擊特定學習內容, **Then** 系統顯示該學習內容的詳細資訊
4. **Given** 使用者在學習專區頁面, **When** 使用搜尋或分類功能, **Then** 系統過濾並顯示符合條件的學習資源

### User Story 5 - 交易查詢功能 (Priority: P2)

使用者需要能夠查詢自己的交易記錄，了解投資交易歷史。

**Why this priority**: 交易查詢是投資服務的核心功能，讓使用者能夠追蹤自己的交易活動。這是重要的功能，但可以在側邊欄導航建立後再實作。

**Independent Test**: 可以透過以下方式獨立測試：使用者點擊側邊欄的「交易查詢」，進入交易查詢頁面，查看交易記錄列表，可以使用篩選或搜尋功能找到特定交易。此功能可獨立運作，提供完整的交易查詢價值。

**Acceptance Scenarios**:

1. **Given** 使用者已登入, **When** 點擊側邊欄「交易查詢」, **Then** 系統導向交易查詢頁面
2. **Given** 使用者在交易查詢頁面, **When** 查看交易記錄, **Then** 系統顯示交易記錄列表，包含交易日期、交易類型、交易金額、交易狀態等資訊
3. **Given** 使用者在交易查詢頁面, **When** 使用日期篩選功能, **Then** 系統顯示指定時間區間的交易記錄
4. **Given** 使用者在交易查詢頁面, **When** 點擊特定交易記錄, **Then** 系統顯示該交易的詳細資訊
5. **Given** 使用者在交易查詢頁面, **When** 使用搜尋功能, **Then** 系統顯示符合搜尋條件的交易記錄

---

### User Story 6 - 個人資料功能 (Priority: P2)

使用者需要能夠查看和編輯自己的個人資料。

**Why this priority**: 個人資料管理是基本功能，讓使用者能夠維護自己的帳號資訊。這是重要的功能，但可以在側邊欄導航建立後再實作。

**Independent Test**: 可以透過以下方式獨立測試：使用者點擊側邊欄的「個人資料」，進入個人資料頁面，查看個人資訊，可以編輯並更新個人資料。此功能可獨立運作，提供完整的個人資料管理價值。

**Acceptance Scenarios**:

1. **Given** 使用者已登入, **When** 點擊側邊欄「個人資料」, **Then** 系統導向個人資料頁面
2. **Given** 使用者在個人資料頁面, **When** 查看個人資訊, **Then** 系統顯示使用者的基本資料（姓名、電子郵件、電話、地址等）
3. **Given** 使用者在個人資料頁面, **When** 編輯個人資料, **Then** 系統允許使用者修改個人資訊並儲存變更
4. **Given** 使用者在個人資料頁面, **When** 更新個人資料, **Then** 系統驗證輸入資料並儲存更新

---

### User Story 7 - 常見問題功能 (Priority: P3)

使用者需要能夠查看常見問題，獲得使用系統的協助。

**Why this priority**: 常見問題提供使用者支援，幫助使用者解決常見疑問。雖然有價值，但可以在其他核心功能完成後再實作。

**Independent Test**: 可以透過以下方式獨立測試：使用者點擊側邊欄的「常見問題」，進入常見問題頁面，瀏覽問題列表，點擊問題查看答案。此功能可獨立運作，提供完整的 FAQ 體驗。

**Acceptance Scenarios**:

1. **Given** 使用者進入應用程式, **When** 點擊側邊欄「常見問題」, **Then** 系統導向常見問題頁面
2. **Given** 使用者在常見問題頁面, **When** 瀏覽問題列表, **Then** 系統顯示常見問題分類與問題列表
3. **Given** 使用者在常見問題頁面, **When** 點擊特定問題, **Then** 系統顯示該問題的詳細答案
4. **Given** 使用者在常見問題頁面, **When** 使用搜尋功能, **Then** 系統顯示符合搜尋條件的問題

### Edge Cases

- 當使用者未登入時，哪些功能需要登入才能存取？哪些功能可以公開存取？
- 當側邊欄在行動裝置上顯示時，如何處理空間限制與使用者體驗？
- 當使用者點擊功能入口但對應頁面尚未實作時，系統如何處理？
- 當側邊欄功能入口需要根據使用者權限動態顯示時，系統如何處理？
- 當使用者同時在多個裝置使用系統時，側邊欄狀態如何保持一致？
- 當側邊欄功能入口需要顯示通知或未讀數量時，系統如何處理？
- 當使用者快速連續點擊多個功能入口時，系統如何處理導航？

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 系統必須在應用程式中顯示側邊欄導航
- **FR-002**: 系統必須在側邊欄中顯示以下功能入口：開戶流程、我的計劃、投資健檢、學習專區、交易查詢、個人資料、常見問題
- **FR-003**: 系統必須在使用者點擊「開戶流程」時，導向首頁/開戶流程頁面
- **FR-004**: 系統必須在使用者點擊「我的計劃」時，導向我的計劃頁面
- **FR-005**: 系統必須在使用者點擊「投資健檢」時，導向投資健檢頁面
- **FR-006**: 系統必須在使用者點擊「學習專區」時，導向學習專區頁面
- **FR-007**: 系統必須在使用者點擊「交易查詢」時，導向交易查詢頁面
- **FR-008**: 系統必須在使用者點擊「個人資料」時，導向個人資料頁面
- **FR-009**: 系統必須在使用者點擊「常見問題」時，導向常見問題頁面
- **FR-010**: 系統必須在使用者位於某個功能頁面時，在側邊欄中高亮顯示對應的功能入口
- **FR-011**: 系統必須支援側邊欄的展開與收合功能（如適用）
- **FR-012**: 系統必須在「我的計劃」頁面顯示使用者的投資組合列表
- **FR-013**: 系統必須在使用者點擊投資組合時，顯示該投資組合的詳細資訊
- **FR-014**: 系統必須在「投資健檢」頁面提供健檢功能，並顯示健檢結果
- **FR-015**: 系統必須在「學習專區」頁面顯示學習資源列表
- **FR-016**: 系統必須在「交易查詢」頁面顯示交易記錄列表，並支援篩選與搜尋
- **FR-017**: 系統必須在「個人資料」頁面顯示使用者個人資訊，並允許編輯
- **FR-018**: 系統必須在「常見問題」頁面顯示問題列表，並支援搜尋
- **FR-019**: 系統必須根據使用者登入狀態，決定哪些功能需要登入才能存取
- **FR-020**: 系統必須在側邊欄中提供清晰的視覺指示，幫助使用者了解當前所在位置

### Key Entities _(include if feature involves data)_

- **使用者 (User)**: 使用系統的使用者，具有個人資料、投資組合、交易記錄等屬性
- **投資組合 (InvestmentPortfolio)**: 使用者的投資組合，包含投資標的、投資金額、投資日期等資訊
- **交易記錄 (Transaction)**: 使用者的交易記錄，包含交易日期、交易類型、交易金額、交易狀態等
- **學習資源 (LearningResource)**: 學習專區的學習內容，包含標題、內容、類型、分類等
- **常見問題 (FAQ)**: 常見問題與答案，包含問題、答案、分類等

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 使用者能夠在 1 秒內看到側邊欄並識別所有功能入口
- **SC-002**: 使用者能夠在 2 秒內從側邊欄導航到任何功能頁面
- **SC-003**: 95% 的使用者能夠成功透過側邊欄找到並存取所需功能，無需額外協助
- **SC-004**: 使用者能夠在 3 秒內從「我的計劃」頁面查看投資組合列表
- **SC-005**: 使用者能夠在 5 秒內完成投資健檢並查看結果
- **SC-006**: 使用者能夠在 3 秒內從「交易查詢」頁面找到特定交易記錄（使用篩選或搜尋）
- **SC-007**: 使用者能夠在 2 秒內從「個人資料」頁面查看個人資訊
- **SC-008**: 90% 的使用者能夠理解側邊欄的導航結構，並知道如何在不同功能間切換
